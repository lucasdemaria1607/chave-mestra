## 1. Filosofia

### O que é o Sistema de Conhecimento

O Sistema de Conhecimento não é uma pasta de anotações. Não é um segundo cérebro genérico. É a espinha dorsal intelectual de um negócio digital — o lugar onde experiência vira inteligência estruturada e inteligência estruturada vira resultado no mundo.

**Tese central:** quem pensa melhor não depende de mais conteúdo. Depende de um sistema que processe o que já consome e transforme em arsenal — conteúdo, copy, ofertas e posicionamento como subproduto natural do próprio pensamento.

### As Duas Chaves

```
INTELIGÊNCIA ──────────────────────────────────────────────────►
  Capacidade de processar, conectar, interpretar e transformar
  informação em visão, decisão e ação.
  "Forjar a própria cabeça."

PERSONALIDADE ──────────────────────────────────────────────────►
  Disposição de agir a partir do que aprendeu, de forma
  particular e insubstituível — marcada pela própria trajetória.
  "O que te torna impossível de copiar."
```

O sistema não escolhe entre as duas. O Sistema de Conhecimento desenvolve Inteligência. A Personalidade emerge de quem você se torna ao longo do processo.

### O Princípio 100% Cérebro + 100% IA

O sistema nunca permite que o usuário pare de pensar. A IA amplifica quem você se tornou. Se você se tornou profundo, ela amplifica profundidade. Se você delegou o pensamento, ela amplifica mediocridade.

**Regra:** toda nota exige reformulação em voz própria. Copiar não conta. Resumir em suas palavras é o mínimo. Formular uma posição é o ideal.

### A Espiral

```
Conhecimento aplicado
        ↓
   Experiências
        ↓
  Erros + Vitórias
        ↓
 Reflexão deliberada
        ↓
Perspectiva singular
        ↓
Posicionamento único
        ↓
 Conteúdo incopiável
        ↓
Audiência que segue VOCÊ
        ↓
 Negócio com moat real
        ↓
 Novas experiências ──────► (reinicia em nível superior)
```

Cada volta da espiral parte de um patamar mais alto. O sistema é o mecanismo que garante que cada volta seja mais densa que a anterior.

## 2. Arquitetura

O sistema opera sobre 4 conceitos fundacionais. São **ARQUITETURA** — permanentes, não mudam. Tudo o mais (técnicas de leitura, memorização, pedagogia) é conteúdo que flui pela arquitetura.

```
┌─────────────────────────────────────────────────────────────┐
│                    ARQUITETURA DO SISTEMA                   │
│                                                             │
│   ZETTELKASTEN        ──  como o conhecimento é REGISTRADO  │
│   (Notas atômicas)        e CONECTADO                       │
│                                                             │
│   SEGUNDO CÉREBRO     ──  como o conhecimento é ORGANIZADO  │
│   (PARA)                  e ACESSADO                        │
│                                                             │
│   MODELOS MENTAIS     ──  como o conhecimento é DESTILADO   │
│   (Frameworks)            e APLICADO — o produto final      │
│                                                             │
│   PROGYMNASMATA       ──  como o conhecimento é EXERCITADO  │
│   (Retórica clássica)     e INCORPORADO no corpo            │
└─────────────────────────────────────────────────────────────┘
```

### Zettelkasten

Sistema de notas atômicas interligadas. Cada nota registra um único insight — não um resumo de livro, não uma lista de pontos. Uma ideia, formulada como posição, conectada a outras.

A lógica: conexões entre notas são mais valiosas que as notas em si. Uma nota isolada é informação. Uma nota conectada a outras 5 é conhecimento. Uma rede de 100 notas conectadas é um sistema de pensamento.

### Segundo Cérebro (PARA)

Organização em 4 categorias:

| Categoria | O que contém | Critério |
|---|---|---|
| **Projetos** | Estudos ativos com prazo e objetivo | Tem começo, meio, fim |
| **Áreas** | Disciplinas de desenvolvimento contínuo | Sem prazo — padrão a manter |
| **Recursos** | Material de referência (Zettelkasten, Modelos Mentais, Guias, Ferramentas) | Útil no futuro |
| **Arquivos** | O que foi encerrado | Não ativo, mas preservado |

No Notion: o hub **Sistema de Conhecimento** organiza tudo via PARA. Recursos contém os bancos de dados reais. O hub exibe views filtradas dessas mesmas bases.

### Modelos Mentais

O produto final do sistema. Um Modelo Mental nasce quando um padrão aparece confirmado em 3 ou mais contextos diferentes — livro, experiência, campanha, observação. É um framework de decisão extraído por indução.

```
Nota 1 (Livro X)
    +
Nota 2 (Campanha Y)    ──►  PADRÃO IDENTIFICADO  ──►  MODELO MENTAL
    +
Nota 3 (Conversa Z)
```

Modelos Mentais são o que diferencia um especialista de um estudante. Um estudante acumula informação. Um especialista opera com modelos.

### Progymnasmata

14 exercícios clássicos de retórica grega para prática deliberada de pensamento e expressão. A ponte entre conhecimento acumulado e capacidade de comunicação. 

Sem prática deliberada, o conhecimento fica em estado latente — você sabe, mas não consegue expressar com clareza, força e precisão. Progymnasmata é o treino que resolve isso.

**Os 14 exercícios (em ordem de complexidade):**
Fábula, Narrativa, Chreia, Provérbio, Refutação, Confirmação, Lugar Comum, Encômio, Vitupério, Comparação, Prosopopeia, Descrição, Tese, Lei.

## 3. Os 5 Campos do Saber

O sistema organiza todo conhecimento em 5 Ciências fundamentais. Nenhuma delas é opcional para um expert digital. Cada uma alimenta capacidades específicas no negócio.

```
┌─────────────────────────────────────────────────────────────────┐
│                     OS 5 CAMPOS DO SABER                        │
│                                                                  │
│  LINGUAGEM ──────────────────────────────── CIÊNCIA-MÃE         │
│  Retórica, Lógica, Hermenêutica, Linguística, Oratória          │
│  → Tudo que você cria, vende e comunica passa por aqui          │
│                                                                  │
│  PSICOLOGIA ─────────────────────────────── CIÊNCIA DO OUTRO    │
│  Cognição, Personalidade, Metacognição, Comportamento           │
│  → Entender a persona, persuasão, tomada de decisão             │
│                                                                  │
│  FILOSOFIA ──────────────────────────────── CIÊNCIA DO SENTIDO  │
│  Epistemologia, Fenomenologia, Estética, Estoicismo             │
│  → Visão de mundo, posicionamento, resiliência                  │
│                                                                  │
│  ARTES ──────────────────────────────────── CIÊNCIA DA FORMA    │
│  Narrativa, Análise Crítica, Expressividade, Estética           │
│  → Storytelling, design, criação com intenção                   │
│                                                                  │
│  ECONOMIA ───────────────────────────────── CIÊNCIA DO MERCADO  │
│  Negócios, Marketing, Branding, Neuromarketing, Análise         │
│  → Estratégia, oferta, escala, posicionamento de produto        │
└─────────────────────────────────────────────────────────────────┘
```

**Os dois eixos de vida:** Psicologia (autodomínio) + Linguagem (expressão). Os demais ampliam. Esses dois são inegociáveis.

### O Ciclo de Estudo de 10 Passos

Para estudar qualquer ciência ou disciplina de forma sistematizada:

```
1. Definir a ciência/disciplina
2. Aprender sua história (contexto + evolução)
3. Listar escolas e correntes principais
4. Listar 5-10 autores fundamentais
5. Listar 5-10 obras de referência
6. Ler (com captura de notas no Zettelkasten)
7. Resumir em voz própria
8. Aplicar (conteúdo, produto, decisão, exercício)
9. Repetir com ciências correlatas
10. Aprofundar intersecções (onde as ciências se encontram)
```

| **Formato** | Select | De onde o insight veio |
| **Argumento** | Texto | O insight em voz própria |
| **Evidência** | Texto | O que gerou este insight |

| **Destino de Uso** | Multi-select | Onde foi usada |
| **Última Revisão** | Data | Quando foi revisada pela última vez |

| **Próxima Revisão** | Fórmula | Calculada automaticamente pela Zona |
| **Recorrência** | Número | Quantas vezes esse tema apareceu |
| **Progymnasmata** | Relação | Exercícios vinculados a essa nota |
| **Produto** | Relação | Produto CM para o qual a nota contribui |

### Tipos de Nota

| Tipo | Quando criar | Formato |
|---|---|---|
| **Flash** | Insight imediato — pode ser relevante ou não | 3-5 linhas, sem refinamento |
| **Literatura** | Ao absorver livro, curso, framework externo | Resumo do conceito + aplicação |
| **Permanente** | Insight validado + conexões identificadas | Título como tese, argumento próprio, links |
| **Projeto** | Insight específico de campanha ou produto | Data + produto + aprendizado |
| **MOC** | Mapa de notas relacionadas (Map of Content) | Nota que organiza outras notas sobre um tema |

### Zonas de Processamento

```
INBOX        ──►  LITERATURA  ──►  PERMANENTE  ──►  ESTRUTURA
(capturada)      (em leitura)     (conectada)      (organizada)

                                        ↓
                                    ANALOGIA         OUTPUT
                                    (padrão         (virou
                                   cruzado)         produto)

                                        ↓
                                    DOMÍNIO
                              (mapa de um campo)
```

| Zona | Significado |
|---|---|
| **Inbox** | Entrou, ainda não foi processada |
| **Literatura** | Em leitura/processamento ativo |
| **Permanente** | Processada, conectada, em voz própria |
| **Estrutura** | Organiza outras notas (MOC) |
| **Analogia** | Padrão identificado em múltiplos contextos |
| **Output** | Virou conteúdo, copy, produto |
| **Domínio** | Mapa de um campo de conhecimento |

### Status do Pipeline

| Status | Significado |
|---|---|
| **Capturada** | Entrada no sistema, aguardando processamento |
| **Em processamento** | Sendo trabalhada ativamente |
| **Conectada** | Tem argumento + conexões, pronta para uso |
| **Publicada** | Virou conteúdo ou output externo |

### Potencial (Escala Botânica)

| Potencial | Significado | Ação |
|---|---|---|
| **Semente** | Ideia bruta, pode virar algo | Aguardar contexto, conectar |
| **Broto** | Conexões identificadas, formato emergindo | Refinar argumento, testar |
| **Árvore** | Insight maduro, pronto para uso | Incorporar em output |
| **Fruto** | Gerou múltiplos outputs | Manter como referência, candidato a MOC |
| **Esgotado** | Já extraiu tudo que podia | Arquivar ou consolidar |

### Estrutura de uma Nota Permanente

```markdown
# [Título formulado como tese ou pergunta — nunca como tópico]

Data: YYYY-MM-DD
Tipo: Permanente
Zona: Permanente
Ciência: [campo(s) relacionado(s)]
Tags: [temas]

## Argumento

[O insight em 2-5 parágrafos. Posição própria, não descrição.
 Escreva como se estivesse explicando para alguém inteligente

## Evidência

[O que gerou este insight: livro X / campanha Y / conversa Z / experiência W]
```

## 5. O Sistema de Revisão (Anki)

### Lógica

Notas não revisadas ficam estagnadas. O sistema precisa trazer de volta à atenção o que está maduro — não aleatoriamente, mas com base em quando foi visto pela última vez e em qual estágio está.

Isso é o que o Anki faz: mostra o cartão certo no momento certo, com intervalo crescente a cada revisão bem-sucedida.

O Zettelkasten agora tem o mesmo mecanismo.

### Como Funciona

Três propriedades trabalham juntas:

```
Última Revisão (data)
    +
Zona (estágio da nota)
    ↓
FÓRMULA calcula:
    ↓
Próxima Revisão (data)
```

**Intervalos por Zona:**

| Zona | Intervalo após revisão |
|---|---|
| Inbox | 1 dia |
| Literatura | 3 dias |
| Permanente | 7 dias |
| Estrutura, Analogia, Output, Domínio | 21 dias |
| Sem revisão alguma (Última Revisão vazia) | Urgente — aparece agora |

### Como Usar no Dia a Dia

```
1. Abrir "⏰ Para Revisar Hoje" (no hub ou na aba do Zettelkasten)
2. As notas aparecem ordenadas por urgência (mais antigas primeiro)
3. Para cada nota:
   a. Ler / reler o argumento
   b. Se faz sentido manter: atualizar "Última Revisão" para hoje
   c. Incrementar "Repetições" em +1
   d. Se processou de verdade: mover Zona para o próximo estágio
   e. Se a nota foi usada: marcar "Incorporada" ✅ + Destino de Uso
4. Notas que não fazem mais sentido: descartar conscientemente
```

### Progressão Natural

```
Nova nota chega → Zona: Inbox
Revisão 1 (dia seguinte) → Processa, move para Literatura
Revisão 2 (3 dias depois) → Refina, conecta, move para Permanente
Revisão 3 (7 dias depois) → Confirma maturidade, atribui Potencial
Revisão 4+ (21 dias depois) → Incorpora ou mantém como referência
```

### O Que Acontece com Notas Puladas

Se você capturou uma nota mas não revisou, a fórmula usa `now()` como base — a nota fica sempre aparecendo como urgente até ser tocada. Isso é intencional: o sistema não deixa notas se perderem silenciosamente.

## 7. O Ciclo da Forja

**Princípio:** capturar é fácil. Assimilar é o trabalho real. Sem varredura periódica, o Zettelkasten vira cemitério de boas intenções.

```
Captura (contínua, sem fricção)
           ↓
Camada 1 — Varredura Semanal ──── 15-30 min
           Inbox → classificar → mover ou descartar
           ↓
Camada 2 — Destilação Quinzenal ── 45-60 min
           Em processamento → refinar → conectar → avaliar potencial
           ↓
Camada 3 — Incorporação Mensal ─── 1-2 horas
           Conectadas → usar em output → marcar destino + Radar
```

### Camada 1 — Varredura Semanal

**Frequência:** toda semana (integrada à Forja Semanal)
**Duração:** 15-30 minutos
**View:** 📥 Inbox Pendente

Checklist:
- [ ] Formato está correto para cada nota?
- [ ] Tipo atribuído (Flash / Literatura / Permanente)?
- [ ] Zona atribuída?
- [ ] Pelo menos 1 Ciência marcada?
- [ ] Pelo menos 1 Tag marcada?
- [ ] Merece processamento? → Status: Em processamento
- [ ] Não merece? → Deletar ou Arquivar
- **Regra:** notas em Inbox há mais de 7 dias → processar ou deletar. Sem exceção.

**Via agente:** `"varre minha inbox"` → lista pendências e sugere classificações.

### Camada 2 — Destilação Quinzenal

**Frequência:** a cada 2 semanas
**Duração:** 45-60 minutos
**View:** 🔥 Em Maturação

Checklist:
- [ ] Reescrever o argumento em voz própria (não é resumo — é posição)
- [ ] Identificar pelo menos 2 conexões com outras notas
- [ ] Preencher campo Evidência
- [ ] Atribuir Potencial (Semente / Broto / Árvore)
- [ ] Verificar Recorrência (tema já apareceu? incrementar)
- [ ] Se pronta → Status: Conectada
- [ ] Atualizar Última Revisão com data de hoje

**Via agente:** `"destila as notas"` → apresenta cada nota em maturação e sugere conexões.

### Camada 3 — Incorporação Mensal

**Frequência:** 1x por mês (integrada à Lua Cheia)
**Duração:** 1-2 horas
**Views:** 💎 Prontas Não Usadas + 🌱 Sementes + ⏰ Revisão Vencida

Checklist:
- [ ] Para cada nota Conectada não incorporada, decidir destino:
  - Vira conteúdo? → `Destino: Conteúdo`
  - Vira copy? → `Destino: Copy`
  - Vira oferta/produto? → `Destino: Oferta` ou `Produto`
  - Vira aula? → `Destino: Aula`
  - Alimenta worldbuilding? → `Destino: Worldbuilding`
  - Exercício de Progymnasmata? → `Destino: Exercício`
  - Insight pessoal? → `Destino: Insight Pessoal`
- [ ] Marcar `Incorporada ✅` + Destino de Uso + linkar Produto se aplicável
- [ ] Sementes: alguma virou Broto? Algum Broto virou Árvore? Reclassificar.
- [ ] Revisão Vencida: atualizar Última Revisão para todas revisadas.
- [ ] Radar: 🎯 Radar por Ciência + 🔁 Temas Recorrentes → identificar clusters, picos, tendências.

**Via agente:** `"incorporação mensal"` ou `"radar do especialista"` → executa análise completa.

### Métricas do Especialista

Calculadas pelo agente sob demanda:

| Métrica | O que revela |
|---|---|
| **Taxa de Assimilação** | Quanto do que consome vira conhecimento real (meta: >40%) |
| **Taxa de Incorporação** | Quanto do conhecimento vira output (meta: >30%) |
| **Inbox Stale Rate** | Varredura semanal falhando? (>50% = sim) |
| **Picos de Interesse** | Para onde a mente está gravitando (últimos 30 dias) |
| **Habilidades Emergentes** | Domínios com crescimento acelerado |
| **Temas Obsessivos** | Tags com Recorrência > 5 — padrões dominantes |
| **Diversidade Cognitiva** | Número de Ciências ativas (ideal: 5; <3 = afunilando demais) |

## 8. Os Rituais

Os rituais são a cadência que garante que o sistema não para. Sem cadência, o sistema morre. Com cadência, cada ciclo adiciona camada.

```
GIRO DA CHAVE ────── diário    ──  captura + foco do dia
FORJA SEMANAL ────── semanal   ──  varredura + planejamento
LUA CHEIA ────────── mensal    ──  destilação + incorporação + radar
```

### Giro da Chave (Diário)

**Quando:** toda manhã, antes de qualquer outra coisa.
**Duração:** 15-30 minutos.

Estrutura mínima:
1. Abrir "⏰ Para Revisar Hoje" — revisar 3-5 notas
2. Capturar 1 insight do dia anterior (se tiver)
3. Definir 1 intenção intelectual para o dia (o que vai consumir/criar)
4. Fechar o sistema e trabalhar

O Giro não é para processar. É para manter o contato — o sistema sabe que você está aqui.

### Forja Semanal

**Quando:** toda semana (dia e horário fixo, sem negociar).
**Duração:** 60-90 minutos.

Estrutura:
1. **Varredura da Inbox** (Camada 1 do Ciclo da Forja) — 15-30 min
2. **Revisão da semana** — o que foi capturado? o que foi produzido?
3. **Planejamento da próxima semana** — o que vai estudar? o que vai criar?
4. **Feedback às rotinas** — algo nas saídas das rotinas de IA merece ajuste?

### Lua Cheia (Mensal)

**Quando:** primeiro dia do mês (ou qualquer data fixa mensal).
**Duração:** 2-3 horas. Não comprime.

Estrutura:
1. **Incorporação Mensal** (Camada 3 do Ciclo da Forja) — 60-90 min
2. **Radar do Especialista** — métricas + tendências
3. **Revisão de Objetivos** — o que evoluiu? o que está parado?
4. **Planejamento do mês** — foco de estudo, ritmo de produção, próximo ciclo
5. **Atualização do sistema** — algum plugin/skill precisa ser evoluído?

## 9. As Rotinas de IA

Rotinas são agentes Claude Code remotos que rodam automaticamente em horários programados. Eles leem os arquivos de configuração do sistema, executam pesquisas e geram outputs que entram nos bancos do Notion.

### Mapa de Rotinas

```
ROTINA               FREQUÊNCIA        STATUS          DESTINO NO NOTION
─────────────────────────────────────────────────────────────────────────
Temas em Alta        Seg/Qua/Sex 6h    🟢 Ativa        Zettelkasten (Inbox)
                     (UTC) = 3h Bahia

Pesquisa de Nicho    Dia 1 e 15, 5h    🟢 Ativa        Zettelkasten (Output)
                     (UTC) = 2h Bahia                  + Tarefas Hub

Accountability       Reativar 25/05    🟡 Pausada      Zettelkasten
de Notas             de 2026

Transformação        Reativar 25/05    🟡 Pausada      Conteúdo (ESTEIRA)
em Conteúdo          de 2026
```

### O Que Cada Rotina Gera

**Temas em Alta** → pesquisa temas virais, eventos recentes, polêmicas e tendências do nicho. Entrega 12-15 temas com ângulo, formato sugerido, janela de oportunidade e gatilho emocional. Cada tema vira uma nota Flash na Zona Inbox do Zettelkasten.

**Pesquisa de Nicho** → mapeamento quinzenal de novos players, movimentos dos concorrentes, linguagem em evolução, brechas e riscos. Vira nota Flash na Zona Output + tarefa no hub com a ação recomendada.

**Accountability de Notas** *(em pausa)* → monitora o ritmo de captura no repositório git. Alerta quando há mais de 7 dias sem commits em `sistema-de-conhecimento/notas/`. Gera cobrança via Zettelkasten.

**Transformação em Conteúdo** *(em pausa)* → pega notas do Zettelkasten e transforma em peças de conteúdo prontas para publicação (roteiro, carrossel, post). Deposita na base Conteúdo da ESTEIRA com status Rascunho.

### Onde Aparecem os Outputs

```
Hub "Sistema de Conhecimento"
    ├── "Visualização de Notas" ── mostra Temas em Alta + Pesquisa de Nicho
    └── "Visualização de Tarefas" ── mostra Ação Recomendada da Pesquisa
```

### O Loop de Feedback

As rotinas melhoram com feedback. O mecanismo:

```
1. Você usa uma saída da rotina e percebe algo:
      "esse tema foi genérico demais"
      "quero mais foco em linguagem de audiência"
      "os temas estão todos cobertos pelos concorrentes"

2. Você cria uma nota no Zettelkasten:
      Tipo: Flash
      Tags: feedback-rotinas
      Título: "Feedback — Temas em Alta — [data]"
      Conteúdo: o que ajustar

3. Você roda o comando: "sincroniza feedback do notion"

4. Claude Code lê a nota e atualiza FEEDBACK-ROTINAS.md no repositório

5. Na próxima execução, a rotina lê o arquivo e aplica o ajuste
```

O arquivo `06-ROTINAS/config/FEEDBACK-ROTINAS.md` no repositório é a memória das rotinas. Ele contém seções para cada rotina com o que está funcionando, o que foi fraco e os critérios de seleção ajustados.

### Como Sincronizar Outputs no Notion

As rotinas geram arquivos `.md` no repositório. Para que apareçam no Notion, rode o comando em qualquer sessão Claude Code:

```
"sincroniza rotinas no notion"
```

Claude Code lê os outputs mais recentes de `06-ROTINAS/outputs/` e escreve nas bases corretas do Notion (Zettelkasten e Tarefas).

## 11. Glossário

| Termo | Definição |
|---|---|
| **Zettelkasten** | Sistema de notas atômicas interligadas. Do alemão: "caixa de fichários". |
| **PARA** | Projetos, Áreas, Recursos, Arquivo — framework de organização do conhecimento (Tiago Forte). |
| **Nota Flash** | Captura imediata, sem refinamento. Primeiro estágio de qualquer nota. |
| **Nota de Literatura** | Nota criada a partir de fonte externa (livro, curso, artigo). |
| **Nota Permanente** | Insight validado, em voz própria, com conexões explícitas. O núcleo do Zettelkasten. |
| **MOC** | Map of Content — nota que organiza e conecta outras notas sobre um tema. |
| **Modelo Mental** | Framework de decisão extraído por indução a partir de padrões confirmados em 3+ contextos. |
| **Progymnasmata** | 14 exercícios clássicos de retórica grega para prática deliberada de pensamento e expressão. |
| **Ciclo da Forja** | Sistema de 3 camadas de revisão periódica do Zettelkasten (semanal, quinzenal, mensal). |
| **Zona** | Estágio da nota no pipeline de processamento (Inbox → Literatura → Permanente → ...). |
| **Potencial** | Maturidade percebida da nota usando metáfora botânica (Semente → Broto → Árvore → Fruto). |
| **Incorporada** | Propriedade que marca se a nota já gerou um output real (conteúdo, copy, produto, etc.). |
| **Recorrência** | Contador de quantas vezes um tema apareceu no sistema — indica padrões obsessivos. |
| **Repetições** | Contador do sistema Anki — quantas vezes a nota foi revisada conscientemente. |
| **Próxima Revisão** | Campo calculado pela fórmula: Última Revisão + intervalo baseado na Zona. |
| **Insumo Biográfico** | Nota do tipo Experiência ou Biografia — arsenal narrativo para conteúdo e copy. |
| **Tesouro dos Erros** | Base de dados de falhas documentadas — o ativo mais subestimado do sistema. |
| **Giro da Chave** | Ritual diário de captura e revisão (15-30 min, toda manhã). |
| **Forja Semanal** | Ritual semanal de varredura e planejamento (60-90 min). |
| **Lua Cheia** | Ritual mensal de destilação, incorporação e planejamento estratégico (2-3 horas). |
| **CCR** | Claude Code Remote — sessão de agente Claude executando remotamente em cron schedule. |
| **FEEDBACK-ROTINAS.md** | Arquivo git lido pelas rotinas antes de gerar qualquer output. A memória das rotinas. |
| **Hub** | Página principal do Sistema de Conhecimento no Notion — dashboard de todas as views. |
| **Linked Database** | View do Notion que aponta para uma base existente sem duplicar dados. |
