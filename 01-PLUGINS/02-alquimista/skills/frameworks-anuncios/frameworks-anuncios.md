# Frameworks de Anúncios

12 frameworks de copywriting selecionados para o sistema Chave Mestra. Os templates usam [PLACEHOLDERS] genéricos — preenchidos com a linguagem, vocabulário e produtos do cliente em questão.

**Critério de seleção:** estrutura + versatilidade + adequação a públicos que buscam profundidade, não simplificação.

---

→ **Modo de operação:** os [PLACEHOLDERS] são preenchidos com a linguagem do cliente — produto, audiência, mecanismo, comunidade. Ver `00-FUNDACAO/MODO-CLIENTE.md` para o protocolo de intake e tabela de equivalências.

---

## Como usar este arquivo

1. Escolher o framework pelo diagnóstico do insumo
2. Preencher os [PLACEHOLDERS] com dados reais do produto e da persona
3. Adaptar o tom: denso, direto, sem simplificação, sem linguagem de coach motivacional
4. Usar como base para post de carrossel (via Chavossel), reel, stories ou copy de anúncio pago

---

## Tabela de Seleção

| Framework | Situação ideal | Canal |
|---|---|---|
| AIDA | Apresentação de produto novo para público frio | Anúncio pago, carrossel de autoridade |
| PAS | Público com dor ativa e específica | Stories, reel, post direto |
| BAB | Transformação de vida/negócio verificável | Carrossel, vídeo de prova |
| DIC | Público que precisa de ruptura para prestar atenção | Reel, anúncio de interrupção |
| PASTOR | Produto complexo, ticket alto, ciclo longo | Carrossel longo, e-mail, página de venda |
| FAB | Produto com entregáveis tangíveis | Post de lista, carrossel de features |
| STAR | Resultado demonstrável por história | Carrossel de caso, storytelling |
| HEART | Público cético, conexão emocional necessária | Stories sequenciais, vídeo de rapport |
| 4P | Copy de venda direta com urgência | CTA final de lançamento, anúncio de fundo |
| Feel-Felt-Found | Quebra de objeção via empatia | DM, stories de resposta, comentários |
| STORYSELLING | Autoridade por narrativa pessoal | Carrossel de origem, vídeo de posicionamento |
| QUEST | Público qualificado mas não decidido | Sequência de e-mail, carrossel de decisão |

## 1. AIDA — Atenção, Interesse, Desejo, Ação

**Estrutura:** Capturar atenção → criar interesse genuíno → acender desejo → converter em ação.

```
[ATENÇÃO]
[Afirmação ou pergunta que interrompe o scroll — específica, sem clichê]

[INTERESSE]
[Por que isso importa para quem está lendo — contexto, dado, mecanismo]

[DESEJO]
[O que muda na vida do Portador quando o problema é resolvido — resultado concreto]

[AÇÃO]
[CTA direto e único — uma ação, um link, uma decisão]
→ [LINK / PALAVRA-CHAVE]
```

**Exemplo de aplicação:**
```
Você está perdendo Portadores não por falta de produto — mas por falta de argumento.

A diferença entre quem converte e quem não converte não está na oferta. Está em como o mecanismo é comunicado.

Quando você aprende a articular o mecanismo real do seu método, a objeção "é caro" desaparece — porque o Portador entende o que está comprando.

É isso que o [NOME DO PRODUTO] entrega.
→ [LINK / PALAVRA-CHAVE]
```

## 2. PAS — Problema, Agitação, Solução

**Estrutura:** Nomear o problema com precisão → amplificar as consequências reais → apresentar a solução como saída natural.

```
[PROBLEMA]
[Nome exato do problema — específico, sem generalização. O Portador precisa se reconhecer imediatamente]

[AGITAÇÃO]
[O que acontece se o problema continua. Não dramatizar — aprofundar. Consequências reais, não hipotéticas]

[SOLUÇÃO]
[Apresentar o produto como a saída lógica — não como a única saída, mas como a mais precisa para esse problema]
→ [LINK / PALAVRA-CHAVE]
```

**Exemplo de aplicação:**
```
Você produz conteúdo há [TEMPO] e os números não crescem proporcionalmente ao esforço.

O problema não é volume. É ausência de estratégia de copy. Sem argumento por trás da criação, você está criando para o algoritmo — não para o Portador. E o algoritmo não compra.

O [NOME DO PRODUTO] resolve exatamente isso: método de copy para quem já cria mas não converte.
→ [LINK / PALAVRA-CHAVE]
```

## 3. BAB — Antes, Depois, Ponte

**Estrutura:** Mostrar o estado atual (antes) → projetar o estado desejado (depois) → apresentar o produto como a ponte entre os dois.

```
[ANTES]
[Descrição precisa e honesta do estado atual do Portador — sem julgamento, com identificação]

[DEPOIS]
[Descrição concreta e verificável do estado após o produto — não genérico, não motivacional]

[PONTE]
[Como o produto conecta os dois estados — o mecanismo, não a promessa]
→ [LINK / PALAVRA-CHAVE]
```

**Exemplo de aplicação:**
```
Hoje: [SITUAÇÃO ATUAL ESPECÍFICA — ex: "você fecha poucos clientes mesmo com audiência crescendo"].

Depois: [SITUAÇÃO FUTURA ESPECÍFICA — ex: "você tem um script de vendas que converte porque parte do mecanismo certo, não de pressão"].

A ponte: [NOME DO PRODUTO] — [DESCRIÇÃO DO MECANISMO EM UMA FRASE].
→ [LINK / PALAVRA-CHAVE]
```

## 4. DIC — Disrupção, Intriga, Clique

**Estrutura:** Quebrar expectativa → criar curiosidade irresistível → levar ao clique / ação.

**Uso principal:** Primeiro slide de carrossel, thumbnail de reel, primeira linha de anúncio.

```
[DISRUPÇÃO]
[Afirmação que contradiz uma crença estabelecida no nicho — específica, com fundamento]

[INTRIGA]
[A promessa implícita de que há algo mais — sem revelar tudo. Deixar a lacuna aberta]

[CLIQUE]
[CTA para fechar a lacuna — leitura do próximo slide, comentário da palavra-chave, acesso ao link]
→ [LINK / PALAVRA-CHAVE]
```

**Exemplo de aplicação:**
```
[DISRUPÇÃO]
Consistência não é o que faz um expert escalar. É o que faz um produtor de conteúdo esgotar.

[INTRIGA]
O que escala é algo que a maioria nunca para para construir — e que não aparece em nenhuma checklist de "como crescer no Instagram".

[CLIQUE]
Comenta [PALAVRA-CHAVE] que eu explico na DM.
```

## 5. PASTOR — Problema, Amplificação, História, Testemunho, Oferta, Resposta

**Estrutura:** O framework mais completo para produto de alto ticket. Funciona como mini página de venda em formato de post longo ou sequência de carrossel.

```
[PROBLEMA]
[Definição precisa do problema central — o que está impedindo o Portador de ter o resultado]

[AMPLIFICAÇÃO]
[Profundidade do problema — o que ele custa, o que ele mantém travado, o que ele compromete no futuro]

[HISTÓRIA]
[Narrativa de transformação — pode ser do próprio expert ou de um Portador. Específica, com contexto real]

[TESTEMUNHO]
[Evidência externa — depoimento, dado, resultado de outro Portador]

[OFERTA]
[O que o produto é, o que entrega, como funciona — descrito com clareza e sem eufemismo]

[RESPOSTA]
[CTA final — uma ação, uma direção, um prazo se houver]
→ [LINK / PALAVRA-CHAVE]
```

## 6. FAB — Features, Advantages, Benefits

**Estrutura:** Traduzir características técnicas em vantagens reais e depois em benefícios para o Portador. Usado quando o produto tem entregáveis tangíveis.

```
[FEATURE / CARACTERÍSTICA]
"[NOME DO PRODUTO] tem [CARACTERÍSTICA ESPECÍFICA]"

[ADVANTAGE / VANTAGEM]
"O que significa que [COMO ESSA CARACTERÍSTICA FUNCIONA NA PRÁTICA]"

[BENEFIT / BENEFÍCIO]
"Para você, isso significa [IMPACTO REAL E ESPECÍFICO NA VIDA/NEGÓCIO DO PORTADOR]"

→ [LINK / PALAVRA-CHAVE]
```

**Exemplo em lista (carrossel de features):**
```
✓ [FEATURE 1] — [ADVANTAGE 1] — para você: [BENEFIT 1]
✓ [FEATURE 2] — [ADVANTAGE 2] — para você: [BENEFIT 2]
✓ [FEATURE 3] — [ADVANTAGE 3] — para você: [BENEFIT 3]

[NOME DO PRODUTO] disponível agora.
→ [LINK / PALAVRA-CHAVE]
```

## 7. STAR — Situação, Tarefa, Ação, Resultado

**Estrutura:** Storytelling de caso real. Mostra transformação com contexto, método e resultado verificável. Forte para prova social e autoridade.

```
[SITUAÇÃO]
[Contexto do Portador / personagem antes — específico, reconhecível]

[TAREFA]
[O que precisava ser feito — o desafio ou objetivo concreto]

[AÇÃO]
[O que foi feito — com o produto, método ou abordagem específica]

[RESULTADO]
[O que aconteceu — dado, marco, mudança verificável e concreta]

→ [LINK / PALAVRA-CHAVE]
```

## 8. HEART — Hook, Empatia, Ação, Resposta, Confiança

**Estrutura:** Para público que precisa de conexão emocional antes de confiar. Funciona bem em stories sequenciais e vídeos de rapport.

```
[HOOK]
[Abertura que captura atenção — pergunta, afirmação ou cena que cria identificação imediata]

[EMPATIA]
[Mostrar que você entende a situação do Portador — de dentro, não de fora. Sem condescendência]

[AÇÃO]
[O que você está oferecendo — o produto, a oportunidade, o próximo passo]

[RESPOSTA]
[CTA claro e único]

[CONFIANÇA]
[Razão para confiar — credencial, resultado, garantia, ou evidência concreta]

→ [LINK / PALAVRA-CHAVE]
```

## 9. 4P — Promise, Picture, Proof, Push

**Estrutura:** Copy de venda direta. Forte para fundo de funil e CTA de lançamento.

```
[PROMISE / PROMESSA]
[O resultado específico que o produto entrega — em uma frase. Verificável e não genérico]

[PICTURE / IMAGEM]
[Como é a vida do Portador depois — detalhes concretos que tornam o resultado tangível e desejável]

[PROOF / PROVA]
[Evidência de que a promessa é real — depoimento específico, dado, resultado anterior]

[PUSH / IMPULSO]
[CTA com urgência real ou razão para agir agora — prazo, vagas, condição especial]

→ [LINK / PALAVRA-CHAVE]
```

## 10. Feel-Felt-Found (Senti, Sentia, Descobri)

**Estrutura:** Framework de empatia para quebra de objeção. Funciona em DM, stories de resposta e comentários. Valida antes de redirecionar.

```
[SENTI / FEEL]
"Entendo como você se sente — [NOMEAR A OBJEÇÃO COM PRECISÃO]."

[SENTIA / FELT]
"Eu mesmo sentia isso [/ outros Portadores sentiam isso] quando [CONTEXTO DE IDENTIFICAÇÃO]."

[DESCOBRI / FOUND]
"O que descobri foi que [REDIRECIONAMENTO BASEADO NO MECANISMO, não na promessa]."

[CTA]
"É por isso que [NOME DO PRODUTO] foi criado do jeito que foi. Se faz sentido, [PRÓXIMO PASSO]."

→ [LINK / PALAVRA-CHAVE]
```

## 11. STORYSELLING — Narrativa → Insight → Produto

**Estrutura:** Contar uma história que naturalmente leva ao produto como conclusão lógica — não como venda. Forte para autoridade e posicionamento.

```
[NARRATIVA — cena de abertura]
[Cena específica — tempo, lugar, situação. Detalhe que cria presença e identificação]

[NARRATIVA — conflito]
[O que aconteceu. O problema, a tensão, o ponto de virada]

[INSIGHT]
[O que foi descoberto. O mecanismo, a verdade, o aprendizado real — não a moralidade genérica]

[PRODUTO como extensão do insight]
[O produto aparece como consequência natural do insight — não como venda abrupta]
"Foi por isso que eu criei / documentei / desenvolvi [NOME DO PRODUTO]."

[CTA]
→ [LINK / PALAVRA-CHAVE]
```

---

## 12. QUEST — Qualificar, Entender, Educar, Estimular, Transição

**Estrutura:** Para público qualificado que ainda não tomou decisão. Leva o lead por um processo de reflexão que culmina na oferta.

```
[QUALIFICAR]
[Definir exatamente para quem este conteúdo é — com precisão que exclui quem não é o perfil]
"Se você [CRITÉRIO DE QUALIFICAÇÃO], este post é para você."

[ENTENDER]
[Mostrar que você conhece a situação deles em profundidade — diagnóstico sem julgamento]

[EDUCAR]
[Entregar uma informação valiosa — o mecanismo ou perspectiva que eles ainda não tinham]

[ESTIMULAR]
[Conectar o conhecimento entregue ao resultado que eles querem — criar movimento]

[TRANSIÇÃO]
[Apresentar o produto como o próximo passo natural — sem forçar]
→ [LINK / PALAVRA-CHAVE]
```

## Notas de Aplicação

**Tom sempre:** denso, direto, sem eufemismo, sem linguagem de coach motivacional. Tom específico ao posicionamento do cliente.

**Placeholders universais — preencher com a linguagem do cliente:**

| Placeholder | O que colocar |
|---|---|
| `[NOME DO PRODUTO]` | Nome exato do produto do cliente — não nome genérico |
| `[MECANISMO]` | O que faz o produto funcionar — específico ao método do cliente |
| `[RESULTADO ESPECÍFICO]` | Verificável e concreto — não genérico |
| `[MEMBRO / PERFIL]` | Como o cliente chama sua audiência — pode ser "Portadores" (CM), "pacientes", "desenvolvedores", "mães", etc. |
| `[COMUNIDADE]` | Nome da comunidade/grupo do cliente — pode ser "A Ordem" (CM), nome do grupo, nome da turma, etc. |
| `[PALAVRA-CHAVE]` | Termo escolhido para ativação ManyChat |
| `[LINK / PALAVRA-CHAVE]` | Destino final: WhatsApp / LP / Checkout |

**Intake de vocabulário:** realizar o protocolo do `00-FUNDACAO/MODO-CLIENTE.md` antes de preencher os placeholders. Os 5 elementos do intake (marca, produtos, audiência, comunidade, voz) substituem todos os placeholders deste arquivo.
