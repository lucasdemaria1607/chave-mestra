# Modo Cliente — Princípio de Operação do Sistema

O sistema Chave Mestra é um ecossistema de coprodução. Sua função é produzir conteúdo, estratégia e operação para qualquer expert, marca ou negócio que o utilize — não apenas para a marca Chave Mestra.

---

## A separação fundamental

| Camada | O que é | Vocabulário permitido |
|---|---|---|
| **Sistema** | SKILL.md, plugin.json, CLAUDE.md, fluxo interno, instruções de operação | Vocabulário CM (Pergaminho, Portal, Forja, Portadores, A Ordem) como referência de implementação |
| **Output do cliente** | Tudo que é gerado e entregue — copy, roteiros, carrosséis, templates, planejamento, ManyChat | 100% na voz e vocabulário do cliente — zero menção a CM, zero menção a Lucas De Maria |

**A marca Chave Mestra é meta-presente como arquitetura. É invisível no output.**

---

## O que NUNCA aparece em outputs para o cliente

- O nome "Chave Mestra" ou qualquer variação
- O nome "Lucas De Maria", "Grigo" ou qualquer referência ao criador do sistema
- Termos proprietários CM (Pergaminho, Portal, Forja, Portadores, A Ordem, Chavideo, Chavossel) — a menos que o próprio cliente seja a marca Chave Mestra
- Referência ao sistema, ao plugin ou à metodologia de origem
- Qualquer dado, preferência ou contexto de outros clientes

---

## O que SEMPRE aparece em outputs para o cliente

- O nome da marca do cliente
- Os nomes exatos dos produtos do cliente
- O termo que o cliente usa para sua audiência
- O nome da comunidade do cliente (se existir)
- O mecanismo central e o vocabulário proprietário do cliente
- O tom de voz, o posicionamento e as referências de universo do cliente

---

## Intake de Contexto do Cliente

Antes de gerar qualquer output para um cliente novo, o sistema precisa de 5 elementos. Estes elementos alimentam TODOS os plugins durante a sessão.

```
CONTEXTO DO CLIENTE

1. MARCA / NOME
   Como o cliente se chama e como se posiciona no mercado.
   Ex: "Dra. Ana Lima — nutrição funcional para executivos"

2. PRODUTOS (listar os 2–4 principais)
   Nome + tipo + descrição de uma linha para cada.
   Ex:
   - "Protocolo Reset" — programa online de 8 semanas (equivale a Portal)
   - "Consultoria Particular" — atendimento 1:1 mensal (equivale a Mentoria)
   - "Guia de Jejum" — material digital (equivale a Pergaminho)

3. AUDIÊNCIA
   Como o cliente chama as pessoas que serve.
   Ex: "profissionais de alta performance", "executivos", "meus pacientes"

4. COMUNIDADE (se existir)
   Nome do grupo, espaço de membros, turma, ou "não tem comunidade".
   Ex: "Grupo VIP WhatsApp", "turma trimestral", "não tem"

5. VOZ DE MARCA
   3 adjetivos que definem o tom + 1 frase que o cliente NUNCA diria.
   Ex: Tom: "técnico, direto, sem rodeios" / Nunca diria: "você merece viver bem!"
```

---

## Fluxo de ativação com cliente externo

```
1. INTAKE (acima) — capturar os 5 elementos antes de qualquer output
   └── Se já passou pelo Oráculo: usar Alma da Persona + SZC como fonte
   └── Se não passou: fazer as 5 perguntas diretamente

2. TODOS OS PLUGINS operam com o contexto do cliente
   └── A voz, o vocabulário, os nomes de produto = os do cliente
   └── A estrutura, o método, a lógica = do sistema CM (invisível)

3. OUTPUT = 100% na identidade do cliente
   └── O cliente pode publicar, usar ou distribuir sem edição de marca
```

## Equivalências de referência

Os plugins CM foram construídos com terminologia própria. Para adaptar a qualquer cliente, usar a tabela de equivalência:

| Termo CM (referência interna) | Equivalente genérico no output |
|---|---|
| Pergaminho | material digital, guia, ebook, método documentado, PDF — ou o nome que o cliente usa |
| Portal | curso online, programa, trilha, formação, acesso — ou o nome que o cliente usa |
| Forja | intensivo, imersão, workshop, masterclass, presencial — ou o nome que o cliente usa |
| Mentoria Individual | mentoria, consultoria, acompanhamento 1:1 — ou o nome que o cliente usa |
| Portadores | alunos, clientes, pacientes, membros, seguidores — ou o nome que o cliente usa |
| A Ordem | comunidade, grupo, turma, espaço de membros — ou o nome que o cliente usa |
| Chavossel | carrossel (o método não aparece no output, só o resultado) |
| Chavideo | roteiro de vídeo (o método não aparece no output) |
| Copy 3x5 / Método Carga | a calibração acontece internamente — o output é só a copy |
| SZC / Alma da Persona | pesquisa de audiência (o método não aparece no output) |

## Quando o cliente É Chave Mestra

Quando Lucas / CM é o cliente da sessão, o glossário CM aplica-se integralmente nos outputs. Não há separação — a marca é o próprio sistema.
