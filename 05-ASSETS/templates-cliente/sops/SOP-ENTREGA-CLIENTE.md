# SOP — Entrega ao Cliente

**Tipo:** Simples (checklist por fase)
**Quando:** ao concluir qualquer fase do projeto
**Tempo estimado:** 30–45 min por entrega
**Plugin responsável:** qualquer plugin que concluiu a fase

---

## Regra de ouro

O documento de entrega usa exclusivamente:
- Linguagem, nomes de produto e tom de voz **do cliente**
- Zero terminologia CM (sem "Dossiê", "Pergaminho", "Chaveiro", "SZC", "Alma da Persona", etc.)
- Zero semáforos operacionais (🔴🟡🟢 são internos)
- Zero notas de método (o método opera invisível)

---

## Checklist por fase

### Fase 0 — Fundação (DNA + Universo de Marca)

- [ ] Extrair do `02-IDENTIDADE/dna-de-marca.md`:
  - Narrativa fundacional
  - 3 valores inegociáveis
  - Voz: 3 adjetivos + frase que nunca diria
  - Léxico proprietário (tabela: termo → significado)
  - Estética (referências visuais)
- [ ] Extrair do `02-IDENTIDADE/mapa-do-universo.md`:
  - 7 ativos da marca (história de criação, credo, ícones, rituais, antagonistas, palavras, líder)
  - BrandScript simplificado (herói, problema, guia, plano, CTA, sucesso, fracasso)
- [ ] Reescrever em linguagem do cliente (sem referências ao framework)
- [ ] Salvar em `08-ENTREGAS/entrega-fase-0.md`
- [ ] Revisar: algum termo CM escapou?

### Fase 1 — Inteligência (Terreno + Persona)

- [ ] Extrair do `03-INTELIGENCIA/relatorio-de-terreno.md`:
  - Panorama do nicho
  - Mapa de concorrentes (posicionamento + força + fraqueza)
  - Brechas identificadas
  - Recomendação de posicionamento
- [ ] Extrair do `03-INTELIGENCIA/alma-da-persona.md`:
  - Perfil do cliente ideal (sem o nome "persona")
  - Jornada emocional (dor → estado atual → transformação desejada)
  - Linguagem real (frases reais do público)
  - Objeção principal
- [ ] Salvar em `08-ENTREGAS/entrega-fase-1.md`

### Fase 2 — Estratégia (Oferta + Campanha)

- [ ] Extrair do `04-ESTRATEGIA/oferta-estruturada.md`:
  - Nome da oferta
  - Promessa central
  - O que está incluído (stack limpo, sem "equação de valor")
  - Garantia
  - Preço
- [ ] Extrair do `04-ESTRATEGIA/mapa-de-campanha.md`:
  - Cronograma visual (tabela por fase)
  - O que publicar em cada fase (resumo)
  - Peças que o cliente precisa fornecer (foto, vídeo, depoimento)
- [ ] Salvar em `08-ENTREGAS/entrega-fase-2.md`

### Fase 3 — Criação (Peças prontas)

- [ ] Listar todas as peças em `05-CRIACAO/` produzidas no ciclo
- [ ] Organizar por formato (roteiros / carrosséis / copy / headlines)
- [ ] Para cada peça: confirmar que está na linguagem do cliente
- [ ] Montar índice das peças com data sugerida de publicação
- [ ] Salvar em `08-ENTREGAS/entrega-fase-3.md`

### Fase 4 — Operação (Calendário + Notion)

- [ ] Exportar calendário do mês de `06-OPERACAO/calendario-[mes].md`
- [ ] Confirmar que o Notion está atualizado (se aplicável)
- [ ] Gerar resumo de publicações do período (o que foi ao ar, o que ficou pendente)
- [ ] Salvar em `08-ENTREGAS/entrega-fase-4.md`

### Fase 5 — Evolução (Relatório pós-ciclo)

- [ ] Extrair de `09-EVOLUCAO/relatorio-[mes].md`:
  - Resultados do período (métricas)
  - O que funcionou (repetir)
  - O que não funcionou (ajustar)
  - Recomendação estratégica para o próximo ciclo
- [ ] Formatar como documento de decisão (não de justificativa)
- [ ] Salvar em `08-ENTREGAS/entrega-fase-5.md`

---

## Checklist final antes de enviar

- [ ] Nome/marca do cliente corretos em todo o documento
- [ ] Nenhum termo CM no documento de entrega
- [ ] Formatação limpa (sem semáforos, sem notas de rodapé internas)
- [ ] Dossiê atualizado com a entrega registrada no §7
- [ ] Versão incrementada no HISTÓRICO DE VERSÕES
