# Criar Produto na Esteira — Protocolo de Criação Completa

A criação de um produto na MINHA ESTEIRA não é só adicionar uma linha. É inaugurar um hub: a página do produto vira o ponto de convergência de todo conteúdo, campanha, entrega, prospecção e métrica relacionados a ele.

Ares entrega um documento formatado com todas as seções preenchidas — pronto para o especialista criar a página no Notion.

---

## Pré-Requisitos

| Pré-requisito | Se não existir |
|---|---|
| Persona mapeada (Apolo) | Criar com persona provisória — marcar como `[sem contexto de persona — a validar]` |
| Oferta esboçada (Atena) | Preencher OFERTA brief com promessa provisória |
| Posicionamento | Opcional — pode ser preenchido depois |

---

## Etapa 1 — Coletar Informações (5 perguntas)

1. **Tipo de produto:** Curso / Mentoria / Workshop / Comunidade / Consultoria / Low Ticket / Isca Digital
2. **Nome provisório** (pode mudar depois)
3. **Preço** (ou faixa: low ticket / mid / high)
4. **Data prevista de lançamento** (ou "a definir")
5. **Formato:** Individual / Grupo pequeno (<30) / Escala (30+)

---

## Etapa 2 — Formato de Output

Ares entrega um documento com todas as seções abaixo preenchidas:

### SEÇÃO: NOME DO PRODUTO
[nome provisório]
Status: Rascunho | Preço: R$[X] | Data de Lançamento: [data ou a definir]

---

### SEÇÃO: OFERTA Brief

- **Mecanismo Único:** [diferencial do produto]
- **Promessa Central:** [transformação em X tempo para Y pessoa]
- **Inimigo Narrativo:** [culpado externo que tem impedido o resultado]
- **N-level do Público:** [N1–N5]
- **Objeções Principais:**
  1. [objeção 1]
  2. [objeção 2]
  3. [objeção 3]

### SEÇÃO: ESQUEMA DO PRODUTO

**Curso:**
```
Módulo 1 — [Nome] → [objetivo de aprendizado] → [duração]
Módulo 2 — [Nome] → [objetivo de aprendizado] → [duração]
Módulo 3 — [Nome] → [objetivo de aprendizado] → [duração]
[Bônus se houver]
Carga horária total: X horas
Formato: Gravado / Ao vivo / Híbrido
Canal de suporte: WhatsApp / Telegram / Plataforma
```

**Mentoria (Grupo ou Individual):**
```
Sessão 0 — Diagnóstico e alinhamento de expectativas
Sessão 1 — [tema] → [entregável]
Sessão 2 — [tema] → [entregável]
Check-ins: [frequência + canal]
Materiais de apoio: [o que é entregue]
```

**Workshop:**
```
Parte 1 — [nome] → [duração] → [entregável]
Parte 2 — [nome] → [duração] → [entregável]
Material de apoio: [templates, PDFs]
Formato: Ao vivo / Gravado
```

**Comunidade:**
```
Frequência de encontros: [semanal / quinzenal / mensal]
Formato dos encontros: [duração + estrutura]
Canal principal: [WhatsApp / Telegram / Discord / Circle]
Conteúdo recorrente: [o que é entregue todo ciclo]
Benefícios de acesso: [o que o membro tem disponível]
```

**Low Ticket / Isca:**
```
Formato: [PDF / Vídeo / Template / Checklist / Mini-curso]
Entregável principal: [o que o cliente recebe]
Tamanho: [páginas / minutos / módulos]
Resultado imediato: [o que o cliente consegue fazer logo após consumir]
```

## Confirmação Final (o que Ares entrega)

```
Produto criado: [nome]
Status: Rascunho
Tipo: [curso / mentoria / workshop / etc.]

✅ OFERTA brief preenchido
✅ Esquema do produto ([tipo]) completo
✅ Tabela de métricas criada (vazia)
✅ Fluxo de entrega estruturado
✅ Estrutura de prospecção pronta
✅ Lista de tarefas com prazos relativos (D-30 a D+21)

Próximos passos:
1. Especialista cria a página na MINHA ESTEIRA no Notion com este conteúdo
2. Preencher OFERTA brief completo com Atena
3. Planejar campanha com Ares → Mapa de Campanha
4. Iniciar prospecção ativa → Ares → Prospecção Ativa
```

---

## Regras

1. Nunca criar produto sem coletar as 5 informações da Etapa 1
2. Tarefas com prazo relativo (D-30, D-20 etc.) — o especialista ajusta para datas reais
3. Todo output entregue formatado — o Notion é responsabilidade do especialista
4. Produto sem promessa central definida = produto sem argumento de autoridade
