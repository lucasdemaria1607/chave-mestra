# Scripts — Criação, Análise e Otimização de Roteiros

Este arquivo cobre três operações de roteiro:
1. **Script Creator** — cria roteiro completo de 60–90s a partir de headline validada
2. **Script Analyzer** — engenharia reversa de roteiros virais
3. **Copy Enhancer** — otimiza roteiro existente para entrega falada

---

## PARTE 1 — SCRIPT CREATOR

### Estrutura do Roteiro (5 Blocos)

**BLOCO 1 — HOOK (0–3s)**
A headline vira fala. Adaptar para linguagem oral — ritmo de conversa, não de texto.
- Máximo 15 palavras faladas
- Deve gerar curiosidade ou identificação imediata
- Nunca revelar a conclusão
- Tom compatível com o posicionamento do expert

**BLOCO 2 — INTENSIFICADOR (3–10s)**
Amplificar a promessa do hook e eliminar a tentação de sair.
- Contraste rápido: "A maioria faz X. Os que crescem fazem Y."
- Prova imediata: "Isso me custou R$30 mil pra descobrir."
- Loop aberto: "E o terceiro ponto é o que muda tudo — mas antes..."
- Espelho emocional: descrever a sensação que o público reconhece

**BLOCO 3 — CONTEÚDO NOTÁVEL (10–50s)**
Entregar valor real que justifique o tempo investido.

Estruturas disponíveis:
- **Lista numerada** (3–5 pontos): "Primeiro... Segundo... Terceiro..."
- **Narrativa pessoal:** história real com arco mini (problema → virada → lição)
- **Desmistificação:** "Todo mundo acha que X. Na verdade, Y. A prova é Z."
- **Passo a passo:** micro-tutorial com resultado tangível
- **Comparação:** antes vs depois, certo vs errado, amador vs profissional

Regras do Bloco 3:
- Cada ponto em no máximo 2 frases
- Dados específicos > afirmações genéricas
- Pelo menos 1 elemento concreto por ponto
- Ritmo: alternar frases curtas com uma frase média a cada 3–4 frases

**BLOCO 4 — APRESENTAÇÃO MAGNÉTICA (50–60s)**
Posicionar o expert como autoridade legítima sem parecer autopromoção.
- Resultado concreto: "Nos últimos 12 meses, X alunos fizeram Y."
- Credencial contextual: conectar experiência ao problema resolvido
- Prova social indireta: "Isso é o que ensino no [produto] — e é por isso que [resultado]."

**BLOCO 5 — CTA (60–90s)**
Um único próximo passo. Nunca dois CTAs.

Tipos por objetivo:
- Engajamento: "Comenta X que eu te mando Y"
- Seguimento: "Segue pra não perder o próximo sobre Z"
- Conversão: "Link na bio — [nome do produto]"
- Comunidade: "Entra no grupo — link nos comentários"

### Processo de Criação

**Passo 1 — Análise da Headline**
- Emoção dominante? (curiosidade, medo, desejo, indignação)
- Promessa implícita?
- Nível de consciência? (N1–N5)

**Passo 2 — Seleção de Estrutura do Bloco 3**
- Headlines de autoridade → narrativa pessoal ou desmistificação
- Headlines de dor → espelho emocional + lista de soluções
- Headlines de curiosidade → revelação progressiva

**Passo 3 — Escrita**
- Ordem: Hook → Intensificador → Conteúdo → Apresentação → CTA
- 150–225 palavras faladas (= 60–90 segundos)
- Linguagem oral: contrações, pausas, ritmo de conversa

**Passo 4 — Checklist**
- [ ] Hook captura em 3 segundos?
- [ ] Intensificador elimina saída?
- [ ] Conteúdo entrega valor real?
- [ ] Apresentação posiciona sem arrogância?
- [ ] CTA único e claro?
- [ ] Tempo dentro de 60–90s?

### Formato de Output — Script Creator

```
SCRIPT CREATOR — [HEADLINE] / [EXPERT] / [NICHO]

ANÁLISE DA HEADLINE:
Emoção dominante: [X]

(texto falado)

```

## PARTE 2 — SCRIPT ANALYZER

Engenharia reversa completa de roteiros virais — tornando visível a arquitetura invisível e extraindo padrões replicáveis.

### Framework de Análise (5 Camadas)

**Camada 1 — Esqueleto Progymnasmata**
- Qual dos 14 exercícios opera como base?
- Há exercícios secundários sobrepostos?
- Como o exercício se manifesta em cada seção?

**Camada 2 — Técnicas Cinematográficas**
- Quais dos 20 mecanismos aparecem na abertura (0–3s)?
- Quais sustentam o desenvolvimento?
- Quais fecham o roteiro?
- Como se combinam e se reforçam?

**Camada 3 — Camadas de Copy**
- **C1 (Emocional/Persona):** qual espelho emocional? qual dor/desejo da Alma da Persona?
- **C2 (Marca/Tribo):** linguagem tribal, símbolos, inimigo nomeado?
- **C3 (Racional/Oferta):** qual objeção foi dissolvida? qual caminho de ação foi facilitado?

**Camada 4 — Gatilhos Psicológicos**
Identificar os 7 primeiros: Recompensa, Reconhecimento, Objeção, Medo, Mistério, Crença, Popularidade.

**Camada 5 — Pergunta Dramática Central**
A pergunta que sustenta todo o roteiro. A resposta a ela É a arquitetura do script.

### Formato de Output — Script Analyzer

```
ANÁLISE CHAVIDEO — [Título/Tema do Roteiro]

VISÃO GERAL
Tipo de roteiro: [Curto 60-90s / Médio / Longo]
Potencial viral: [Alto / Médio / Baixo] — [justificativa]
Dramatic Question central: "[a pergunta que sustenta tudo]"

---

CAMADA 1 — ESQUELETO PROGYMNASMATA
Exercício principal: [nome]
Exercícios secundários: [lista]
Como se manifesta:
- Abertura: [como o exercício abre]
- Desenvolvimento: [como progride]
- Fechamento: [como resolve]

---

CAMADA 2 — TÉCNICAS CINEMATOGRÁFICAS
| Técnica | Onde aparece | Como funciona neste roteiro |
[preencher todas as técnicas presentes]

Combinação principal:
[Técnica A] + [Técnica B] + [Técnica C] = [efeito conjunto]

---

CAMADA 3 — CAMADAS DE COPY
C1 — Espelho emocional: [o que espelha] | Inferno ativado: [dor] | Paraíso prometido: [transformação]
C2 — Linguagem tribal: [expressões] | Inimigo: [quem/o quê] | Aliado: [quem/o quê]
C3 — Fricção removida: [objeção dissolvida] | Caminho de ação: [o que é facilitado]

---

CAMADA 4 — GATILHOS
| Gatilho | Presente? | Como aparece |
[preencher os 7]
Gatilho dominante: [X] — por que funciona: [1-2 linhas]

---

CAMADA 5 — RITMO E PROGRESSÃO
0–3s: [como a tensão sobe]
3–30s: [progressão]
30–60s: [pico e resolução]
Ponto mais fraco: [onde poderia perder — e por que não perdeu]

---

TEMPLATE REPLICÁVEL
Esqueleto base: [Exercício Progymnasmata]
Combinação: [Técnica A] + [Técnica B] + [Técnica C]
Padrão de abertura: "[padrão extraído]"
Progressão: [estrutura do desenvolvimento]
Fechamento: [como resolver com payoff]

---

DIAGNÓSTICO FINAL
Por que virou viral: [3–5 elementos específicos]
O que ensina sobre criação: [insight replicável em 2–3 linhas]
```

### Anti-Padrões a Sinalizar

- Twist sem preparação prévia — falha técnica
- Payoff desproporcional à promessa
- Pergunta Dramática que desaparece no meio
- Retenção por confusão (≠ mistério)
- CTA desalinhado com o conteúdo

## PARTE 3 — COPY ENHANCER

Transforma roteiros escritos em performance falada — cortando o que não funciona em voz alta, fortalecendo o que funciona, criando ritmo que segura a atenção em movimento.

**Princípio:** um roteiro que lê bem no papel frequentemente morre na câmera. Otimizar para o ouvido, não para o olho.

### Distinção: Escrita vs Fala

| Copy Escrita | Copy Falada |
|---|---|
| Frases completas | Fragmentos permitidos |
| Gramática formal | Gramática conversacional |
| Lida visualmente | Soa natural |
| Parágrafos densos | Unidades curtas e diretas |
| Verbos podem ser fracos | Cada verbo carrega peso |
| Adjetivos aceitáveis | Substantivos e verbos fazem o trabalho |

### Protocolo de Otimização (6 Passos)

**Passo 1 — Auditoria Estrutural**
Eliminar: marcadores de seção vazados ("GANCHO:", "INTENSIFICADOR:"), bullet points no roteiro final, cabeçalhos dentro da narrativa.

**Passo 2 — Eliminação de Verbosidade**
- Remover preenchimento: "É importante notar que...", "Como podemos observar..."
- Cortar qualificadores redundantes: "muito", "bastante", "realmente" sem função
- Comprimir: se uma ideia toma 30 palavras, encontrar a versão de 15

*Antes:* "Uma coisa que é muito importante que você entenda é que o problema que a maioria das pessoas enfrenta não é falta de conhecimento."
*Depois:* "O problema não é falta de conhecimento."

**Passo 3 — Fortalecimento de Verbos e Substantivos**
- "faz com que seja difícil" → "paralisa"
- "tem um efeito positivo em" → "transforma"
- "o processo de desenvolvimento" → "a forja"

**Passo 4 — Ritmo e Cadência**
- Variação de tamanho: Longa. Curta. Muito curta. Impacto. Depois média.
- Repetição para ênfase: "Não é sobre X. Nunca foi sobre X."
- Tricolon: "Estuda. Testa. Refina."
- Anáfora: "Você que... Você que... Você que..."

**Passo 5 — Carga Cognitiva**
Um parágrafo = uma ideia. Verificar se a progressão é lógica quando OUVIDA em sequência.

**Passo 6 — Calibração de Duração**
- 60s ≈ 150–180 palavras
- 90s ≈ 225–270 palavras
- 3min ≈ 450–540 palavras

Se longo: cortar redundância, não substância.
Se curto: aprofundar o Conteúdo Notável — não encher artificialmente.

### Formatos de Pedido Comuns

| Pedido | O que fazer |
|---|---|
| "Deixa mais pesado / impactante" | Fortalecer verbos, cortar qualificadores, adicionar tricolon, inserir pausas estratégicas |
| "Deixa mais leve / conversacional" | Fragmentos, contrações, perguntas retóricas, frases mais curtas |
| "Tá longo demais" | Uma ideia por parágrafo; cortar qualquer parágrafo com duas; remover preenchimentos |
| "Tá curto demais" | Aprofundar Conteúdo Notável — especificidade, dado, beat narrativo |
| "Não tá fluindo" | Ler em voz alta, marcar tropeços, reestruturar essas frases para ritmo falado |

### Formato de Output — Copy Enhancer

```
SCRIPT OTIMIZADO — [Título/Tema]

Versão: Original → Otimizada
Duração estimada: [X segundos]
Modificações principais: [lista de 3–5 ajustes feitos]

---

[SCRIPT COMPLETO — TEXTO CORRIDO SEM MARCAÇÕES]

---

NOTAS DE ENTREGA (opcional)
- Onde pausar para impacto
- Onde acelerar o ritmo
- Onde olhar diretamente para a câmera
- Onde gesto ou expressão amplifica a palavra
```

### Checklist de Qualidade Pós-Otimização

- [ ] Zero marcadores estruturais no texto
- [ ] Lido em voz alta em uma passagem sem tropeços
- [ ] Cada frase justifica sua existência
- [ ] Ritmo varia (sem monotonia)
- [ ] Duração bate com o alvo
- [ ] Pergunta Dramática intacta
- [ ] Payoff proporcional à tensão gerada
